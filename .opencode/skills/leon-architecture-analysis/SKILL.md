## Objective
Understand and document system architecture.

## When to use
- When starting a new feature
- When investigating cross-module issues
- During architectural reviews

## Procedure
1. Map module boundaries
2. Analyze dependencies between modules
3. Detect coupling and circular dependencies
4. Review contracts/interfaces
5. Identify responsibility overlaps
6. Propose incremental improvements

## Output
- Architecture diagram (textual)
- Dependency list
- Risk areas identified
- Improvement recommendations
